# 齿轮需求能力

当用户请求齿轮、直齿轮、齿轮轮、传动齿轮、齿轮副，或明显需要围绕节圆分布齿形的零件时，使用本参考文档。

## 需求智能体需要知道什么

系统中有一个可复用的直齿轮标准件。齿轮请求可以作为标准件驱动的零件完成，不需要从零建模每一个齿。

默认源资产：

```text
D:\a_src\python\sw_agent\standard_swpart\gear\spur_gear.SLDPRT
```

## 可编辑主动参数

将这些参数作为用户可见参数或假设补全参数暴露：

- `M`：模数，单位 mm。
- `Z`：齿数。
- `B`：齿宽，单位 mm。
- `Alpha`：压力角，单位度。
- `Hax`：齿顶高系数。
- `Cx`：顶隙系数。

这些参数进入代码阶段时会通过 SolidWorks 方程式修改。需求输出中的 `editable_parameters` 可以使用数值，但建模代码必须把带单位目标写成带单位字符串：`M`、`B` 使用 `"mm"` 后缀，`Alpha` 使用 `"deg"` 后缀；`Z`、`Hax`、`Cx` 为无量纲参数。

如果值缺失，把缺失信息写入 `assumptions`，并选择保守默认值。常用默认值：

```json
{"M": 2, "Z": 30, "B": 10, "Alpha": 20, "Hax": 1, "Cx": 0.25}
```

## 支持的二次修改

只有当用户明确提出，或装配协议明确需要时，才把以下内容加入 `secondary_modifications`。不要因为“齿轮通常有孔”就默认添加中心通孔、键槽、轮毂或减重孔。

- 修改齿轮全局变量，例如 `Z`、`M` 或 `B`。
- 添加中心通孔。
- 添加轴孔、轮毂切除、类键槽、减重孔、倒角或安装特征。
- 添加命名接口。独立齿轮零件只补用户明确要求的功能接口；齿轮副或装配任务才补装配用轴线、齿轮中面、前/后端面、孔内圆柱面和节圆相关参考。

## 需求输出指导

对于齿轮零件，在 `part_spec` 中包含：

- `part_type`: `spur_gear`.
- `asset_binding`：源资产路径，以及是否需要物化到工作区。
- `editable_parameters`：上述主动参数。
- `secondary_modifications`：用户明确要求或装配协议必需的二次修改。
- `interfaces`：稳定接口名。独立零件可只包含用户需要的功能接口，例如中心孔任务中的 `shaft_axis`、`bore_cylindrical_face`；装配任务再加入 `front_face`、`back_face`、`gear_mid_plane` 等后续配合会引用的接口。

如果用户没有要求中心孔，独立零件任务不要生成 `bore_cylindrical_face`，也不要添加中心通孔切除。是否定义 `shaft_axis` 取决于用户或装配规划是否会引用旋转轴。

对于齿轮副或类似齿轮箱的请求，使用 `request_type = assembly`，每个唯一齿轮作为一个零件，通过 `instances` 和 `constraints` 表达装配关系。
