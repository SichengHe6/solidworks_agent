# 零件静态一致性校验规则

该文档由 v2 `PartValidationAgent Knowledge Base` 归纳而来。

## 校验职责

静态校验不是运行时仿真，也不是几何精确验证。目标是判断生成代码是否大概率能支撑后续装配继续进行。

## 与零件规格一致

检查代码是否覆盖：

- 零件名称语义。
- 主体形状路线，例如拉伸、旋转、扫描。
- 关键尺寸。
- 主要孔槽、切除、圆角、倒角。
- 需要的功能性接口。独立单零件任务中的接口名通常只是后续复用建议，不等同于装配必需接口。

如果代码建的是完全不同的结构，应判 `pass=false`。

## 与完整装配规划一致

仅当 `full_plan.request_type == "assembly"` 时，才把本节作为硬性检查。独立 `part` 任务不应因为缺少装配面、装配轴或装配点而判失败。

重点检查：

- 该零件在装配中需要的接口是否被实现。
- 接口名称是否与装配规划一致。
- 轴/面的方向关系是否明显冲突。
- 保存路径是否能被后续装配引用。

如果该零件被多个实例复用，应检查：

- 代码是否仍然只生成一个可复用零件模型。
- 该模型是否覆盖所有实例后续需要引用的接口。
- 是否遗漏某个实例专用但几何上仍应由同一零件提供的接口。

## 输出路径

代码必须最终保存到输入 JSON 提供的 `model_file` 对应目标位置或固定工作区下的等价输出路径。

允许使用相对路径、路径变量、`os.path.join`、`Path(...)` 等方式实现，不要求完整绝对路径字面量直接出现。

如果代码把文件保存到工作区外的错误位置，应判失败。

## 高风险问题

- mm/cm 与 m 混淆。SolidWorks Python API 的草图尺寸、拉伸深度、切除深度、参考面偏移和参考轴端点都必须使用米。
- `set_global_variable` 修改带单位方程式目标时缺少单位后缀，或错误地把方程式值换算成米。应按原方程单位写字符串：原方程是 mm 就写 `"2mm"`、`"10mm"`，原方程是 cm 就写 `"2cm"`，角度写 `"20deg"`；`Z`、`Hax`、`Cx` 等无量纲参数才使用裸数字字符串。
- 标准直齿轮模板从 `Z=0` 起沿正 Z 方向拉伸到齿宽 `B`，中面在 `B/2`。不要因为它不是双向拉伸或中心不在 `Z=0` 而判失败。
- 参考轴端点距离数量级明显错误，例如为 10mm 宽的零件创建 `[0,0,0]` 到 `[0,0,10]` 的轴。这会拉大截图包围盒并误导动态反馈。
- 装配任务中缺少命名参考轴或命名参考面。独立零件任务中，如果几何本身完整，不要因缺少这些装配接口单独判失败。
- 接口名与规划 JSON 不一致。
- `revolve` / `revolve_cut` 缺少构造线轴。
- `sweep` 缺少路径或轮廓。
- `shell` 没有可选开口面信息。
- `fillet` / `chamfer` 选择点可疑且没有依据。
- 调用当前封装里没有稳定支持的高级功能。
- 对齿轮标准件通孔切除，如果代码使用 `SelectByID2`、`InsertSketch2`、`sgear.sw_doc` 或 `sgear.sw_instance`，应判为代码实现风险；建议回退到封装级三步流程 `insert_sketch_on_plane`、`create_circle`、`extrude_cut`。

## `pass` 判定

可通过：

- 代码整体与规划一致。
- 路径正确。
- 接口大体齐全。
- 轻微瑕疵不影响后续装配。

不可通过：

- 结构明显不对。
- 尺寸体系明显错误。
- 未保存到指定路径。
- 装配任务缺少关键装配接口。
- 明显会导致后续装配失败。

`feedback` 应保持 1 到 2 句，直接说明最关键问题。
