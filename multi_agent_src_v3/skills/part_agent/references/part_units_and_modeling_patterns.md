# 零件单位与建模套路

该文档由 v2 `PartModelingAgent Knowledge Base` 归纳而来。

## 单位规则

底层封装在新建文档后统一设置为米制。

- 代码中所有建模尺寸都应使用米。
- 如果输入尺寸是 mm 或 cm，必须先换算为 m。
- 不要把 `10 mm` 直接当作 `10 m` 传入 API，也不要把 `2 cm` 直接当作 `2 m` 传入 API。
- 需求和规划中可以保留工程可读的 mm 表达，但代码必须换算。

例外：`set_global_variable` 修改 SolidWorks 方程式时，值应遵循方程式目标自身的原始单位，而不是一律换算成米。

- 对带长度单位的方程式目标，传入带单位字符串，并尽量沿用该方程原本单位，例如 `set_global_variable("M", "2mm")`、`set_global_variable("B", "10mm")`；如果模板原方程使用 cm，则写 `"2cm"`。
- 对带角度单位的方程式目标，传入带角度单位字符串，例如 `set_global_variable("Alpha", "20deg")`。
- 对无量纲目标，传入裸数字字符串，例如 `set_global_variable("Z", "24")`、`set_global_variable("Hax", "1")`。
- 不要把带单位方程式目标写成 `set_global_variable("M", "2")` 或 `set_global_variable("B", "10")`。

### 参考轴和截图包围盒

参考轴端点必须使用米，并且长度应接近零件实际尺寸。轴只需要定义方向，不需要很长。

- 对 10mm 厚的齿轮，轴可以使用 `[0, 0, 0]` 到 `[0, 0, 0.01]`。
- 对 50mm 级零件，参考轴长度通常用 `0.02` 到 `0.1` 这类数量级即可。
- 不要因为把 mm 当 m 而创建 `[0, 0, 0]` 到 `[0, 0, 10]`、`[0, 0, 100]` 这类巨大参考轴；这会显著放大视图包围盒，导致截图反馈误判模型很小或不可见。

## 稳定建模套路

- 主体：中心矩形、圆或多边形草图 + `extrude`。
- 回转体：剖面 + 构造线 + `revolve`。
- 孔槽：在已有面对应平面建草图 + `extrude_cut`。
- 对称结构：使用中心草图和全尺寸参数，减少方向歧义。
- 装配接口：只有装配规划会引用时，创建命名参考面和命名参考轴；独立单零件任务不需要为了静态检查强行创建装配接口。

## 特征前置条件

### `revolve` / `revolve_cut`

- 草图中必须有闭合轮廓。
- 草图中必须有一条构造线作为旋转轴。

### `sweep`

- 必须同时有轮廓草图和路径草图。
- 路径连续，轮廓与路径起点关系合理。

### `shell`

- 必须提供用于选择开口面的面上点坐标。

### `fillet` / `chamfer`

- 边上点或面上点坐标要足够准确。
- 半径和倒角距离不要大到产生几何干涉。

## 代码风格

推荐结构：

1. 导入封装。
2. 解析输入参数和路径。
3. 创建应用和零件文档。
4. 分步骤建模，并打印关键步骤日志。
5. 如果是装配零件或用户明确要求接口，创建参考面和参考轴接口。
6. 保存到指定路径。
7. 对关键失败点进行基本判空或异常处理。

必须避免：

- 忘记保存到输入指定路径。
- 混淆 mm/cm 与 m。
- 在需要旋转轴时没画构造线。
- 依赖未确认存在的高级能力。
- 输出伪代码而不是可执行代码。
- 在装配任务中不体现接口名称，导致后续装配无法引用。独立零件任务不要为满足这条而添加无用的大尺寸参考几何。
