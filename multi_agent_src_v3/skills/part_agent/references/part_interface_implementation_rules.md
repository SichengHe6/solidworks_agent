# 零件接口实现规则

该文档由 v2 `PartModelingAgent Knowledge Base` 归纳而来。

## 复用零件模板

先区分任务类型：

- 独立 `request_type=part` 任务中，接口通常只是可读性或后续复用建议；不要为了静态检查强行创建无用装配接口。
- `request_type=assembly` 的零件子任务中，装配约束实际引用的接口必须实现并保持命名一致。

如果完整装配规划中显示该零件对应多个 `instances`：

- 实现的是一个可复用的统一零件模板。
- 接口应覆盖所有实例实际需要的接口，并保持命名稳定。
- 不要为每个实例各写一份几何几乎相同的代码。
- 只有实例对应的几何特征本身不同，才需要反馈装配规划拆件可能有误。

## 面接口

优先通过命名参考面暴露面接口。

推荐命名：

- `mount_face_top`
- `base_face_bottom`
- `front_face`
- `back_face`
- `gear_mid_plane`

## 轴接口

优先使用 `create_axis(pt1, pt2, axis_name=...)`。

轴方向应与规划中的 `direction_relation` 一致。

轴端点使用米，并且长度应接近零件实际尺寸。轴只需要表达方向，不需要很长；例如 10mm 厚零件使用 `[0, 0, 0]` 到 `[0, 0, 0.01]` 即可。不要把 mm 值直接当 m 传入，避免参考轴过长影响截图包围盒。

推荐命名：

- `main_axis_z`
- `shaft_axis`
- `hole_axis_1`
- `planet_axis_1`

## 点接口

当前没有成熟的命名点封装。需要点位语义时：

- 用变量保留关键点坐标。
- 用这些点创建参考轴。
- 在日志或 sidecar 中标注点位用途。

## 保存与装配可引用性

装配阶段依赖零件文件路径、参考面名称和参考轴名称。对装配零件代码必须确保：

- 最终保存 `.SLDPRT`。
- 接口名称与规划 JSON 一致。
- 日志中打印关键接口创建结果。
