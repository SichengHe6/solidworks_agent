# 静态校验规则

- 检查语法和保存路径使用。
- 对独立 `request_type=part` 的任务，接口名称只作为可读性和后续复用建议，不得因为缺少装配接口而直接判失败。
- 只有 `request_type=assembly` 的零件子任务，才检查装配规划实际引用的必要接口是否出现在代码或注释中。
- 检查生成特征是否匹配零件规格和装配复用需求。
- 检查代码是否把 mm/cm 错当成 m；SolidWorks Python 建模 API 的长度值必须是米。
- 检查参考轴、参考面偏移、草图尺寸、拉伸/切除深度是否处于合理数量级。比如 10mm 应写成 `0.01`，不能写成 `10`；参考轴端点距离应接近零件尺寸，不应因为单位错用而变成数米或数十米。
- 检查 `set_global_variable` 是否对带单位方程式目标保留原方程单位后缀；齿轮模板中 `M`、`B` 应使用 `"2mm"`、`"10mm"` 这类值，`Alpha` 应使用 `"20deg"`，而 `Z`、`Hax`、`Cx` 使用裸数字字符串。
- 检查重复实例是否复用同一个零件模板，而不是按实例重复建模。
- 检查 `revolve` 是否具备构造线轴、`sweep` 是否具备轮廓和路径、`shell` 是否具备开口面选择点。
