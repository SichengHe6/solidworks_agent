# 零件智能体技能

## 角色

零件智能体负责把详细零件规格转换为可执行的 SolidWorks Python 代码，校验代码是否符合设计意图，运行动态执行流程，并在失败时修复建模思路或代码实现。

## 技能加载策略

只加载与当前零件类型、特征标签、失败类型或修复目标相关的参考文档。

当 `part_type`、特征标签、资产元数据或修复反馈中出现齿轮、直齿轮、齿数、模数、孔、轴、键槽或标准件时，加载齿轮参考文档。

## 核心职责

1. 在代码生成前先思考建模策略。
2. 生成或局部修复 Python 代码。
3. 对照零件规格执行静态一致性校验。
4. 通过工具流程运行动态执行、截图和结果检查。
5. 判断失败来自建模思路还是代码实现。
6. 将最终建模思路和最终代码保存到零件输出目录。
7. 对标准件驱动的齿轮，先把源文件物化到工作区，再通过全局变量修改参数，并在工作副本上添加二次特征。
8. 修改 SolidWorks 方程式全局变量时，按方程式目标原本的单位写带单位字符串；例如原方程是 mm 就写 `"2mm"`、`"10mm"`，原方程是 cm 就写 `"2cm"`，角度写 `"20deg"`，无量纲变量才使用裸数字字符串。
9. SolidWorks Python 建模 API 的长度输入统一使用米；草图、拉伸、切除、参考面偏移、参考轴端点都不要把 mm/cm 数值直接传入 API。
10. 独立单零件任务不应为了通过静态检查而强行补装配接口；只有装配规划明确会引用某面/轴/点时，才把这些接口作为必须实现项。
