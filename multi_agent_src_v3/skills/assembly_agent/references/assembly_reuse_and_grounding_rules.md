# 装配复用与基准件规则

该文档由 v2 `AssemblyPlanningAgent` 和 `AssemblyAgent` 知识库归纳而来。

## 重复零件复用

如果多个实例引用同一个 `part_id`：

- 必须复用同一个零件文件路径。
- 通过多次 `add_component(...)` 插入得到多个组件实例。
- 用 `instance_id -> comp_name` 区分这些实例。
- 不要假设每个实例都对应一个独立 `.SLDPRT`。
- 不要只靠 `part_id` 区分重复件。

## 约束解析顺序

当存在重复件时，约束解析应遵循：

1. 根据 `source_instance_id` / `target_instance_id` 找到组件实例。
2. 根据 `source_interface` / `target_interface` 找到接口名称。
3. `source_part_id` / `target_part_id` 只作为校验和辅助信息。

## 基准件处理

装配规划里若有基准件：

- 先插入该零件。
- 视作固定参考。
- 其他零件相对它配合。

如果约束里出现 `GROUND`：

- 表示该接口相对装配全局基准或首个固定件建立关系。
- 代码中应优先通过首个基准件承接这些关系，或按装配框架显式固定该基准件。

## 推荐装配顺序

1. 基准件入装配并固定。
2. 与基准件先做主安装面重合。
3. 再做主轴同心或次级面重合。
4. 最后补充防转、定向和间距类约束。

## 当前系统限制

当前最稳定能力主要是：

- 插入组件。
- 平面配合。
- 轴配合。

不要默认承诺：

- 高级运动副。
- 限位配合。
- 齿轮/螺旋/路径等高级配合。
- 自动求解复杂过约束问题。
